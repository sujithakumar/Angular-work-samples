Introduction

Stable version : 10.1.3 / 23 September 2020

  • Angular is a TypeScript-based open-source front-end platform that makes it easy to build applications with for web/mobile/desktop.
  • Angular is mainly used for developing single-page applications( The single page application is a web application or website that interacts with the user by dynamically rewriting the current page, rather than loading entire new pages from the server)
  • Angular is a completely revived component-based framework in which an application is a tree of individual components.

Features /Pros of Angular

  • Data Binding
  • Angular CLI
  • Lazy Loading
  • Deep Linking
  • Cross-Platform
  • Declarative templates
  • Dependency injection
  • MVC Architecture
  • High Speed
  • Better Error Handling

Cons of Angular

When to use Angular? 

        • You’re designing a progressive web application (PWA)
        • You’re creating a large enterprise website or another large-scale project that involves a complex infrastructure
        • You develop website applications that rely on dynamic content
        • You’d like to upgrade outdated website application designs
        • You’ve been hired to improve the user experience for a large website by initiating a major redesign

Although the platform has its share of cons, Angular is a full-featured and dynamic framework. And its usability, flexibility, and maintainability makes it unique and provides chances to create excellent and successful web-based applications.
